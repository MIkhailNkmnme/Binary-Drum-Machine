# Zerkalius: Genesis — правила работы

## Какой файл редактировать
Рабочий файл — только `Zerkalius-genezis.html`. Не трогай другие файлы
(`banks_data.js`, `_updater.bat`, `_sync_backup.bat` и т.д.), если не попросили отдельно.

## Версия и архивирование
В файле есть маркер версии (около строки 672):
`v0.NNN Ultra-Save` (например `v0.245 Ultra-Save`).

При КАЖДОМ изменении `Zerkalius-genezis.html`, ПОСЛЕ того как код-правки внесены:
1. Увеличь номер версии в маркере на 1 (`v0.245` → `v0.246`).
2. Скопируй готовый файл (уже с новым номером в маркере) в папку `archive/`
   под именем `Zerkalius-genezis-NNN.html`, где `NNN` — этот новый номер
   версии (например `v0.246` → `archive/Zerkalius-genezis-246.html`).
   Архив хранит версию ПОСЛЕ правки, номер архива = номер маркера в файле
   на момент копирования. Не пытайся восстанавливать/реверсить состояние
   до правки — просто copy текущего файла.
3. Если файл с таким номером в `archive/` уже существует (несколько правок
   без бампа версии), добавь свой суффикс: `Zerkalius-genezis-245b.html`,
   `-245c.html` и т.д.

## Никакой лишней возни
Не устраивай проверки синтаксиса через node/python/JScript, не извлекай скрипт
во временные файлы, не пиши diagnostic-скрипты "на всякий случай". Просто:
правка в HTML → бамп версии → копия в archive/, без `cd` в команде копирования
(рабочая директория и так сохраняется между командами). Если нужна проверка —
попроси пользователя открыть файл в браузере и посмотреть консоль.
